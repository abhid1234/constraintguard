The problem — "ConstraintRot"

Compaction is lossy, and constraints are the first casualty.

To fit a growing conversation into a fixed context window, agents summarize their own history. That summary keeps the gist — and quietly drops the rules. The research (arXiv:2606.22528) measured violation rates climbing from 0% to as high as 59% after a compaction. Scroll to watch it happen to a single constraint — then reproduce the numbers yourself below.

STEP 01

You set the rules.

At the start of a session you declare the constraints the agent must respect. While they sit in the context window, the agent follows them reliably — 0% violations.

STEP 02

The session grows.

Turn after turn, user messages, tool calls, and results pile up. The window fills toward its token limit.

STEP 03

The window fills up.

To keep the session alive, the agent has to reclaim space. Something has to give — and the model decides what.

STEP 04

Compaction fires — and the rules vanish.

The history is summarized into a compact blurb. The summary keeps the task’s gist… and silently drops the constraints. The paper calls this Governance Decay.

STEP 05

The agent breaks a rule it can’t see.

Later it takes an action the vanished rule forbade. Nothing errors — the constraint simply isn’t there anymore. Across models, violations jump from 0% to as high as 59%.

STEP 06 · THE FIX

constraintguard pins them back.

Extract the constraints before compaction, re-pin them into the fresh context after. The rules survive the summary, and violations return to 0%.

The evidence — peer-reviewed

This isn't a hunch. It's measured.

The failure has a name in the literature — Governance Decay — a benchmark, and a proposed fix. Before you reach for the solution, here's the problem straight from the research.

arXiv:2606.22528 · June 2026 · Shiyang Chen

Governance Decay: How Context Compaction Silently Erases Safety Constraints in Long-Horizon LLM Agents

"In-context governance constraints that agents reliably obey while visible can be silently removed by compaction, causing the same agent to perform prohibited tool actions later in the session."

Constraint-violation rate across 1,323 episodes · 7 model families. When a constraint survives the summary, violations stay at 0% — when it's dropped, they jump to 38%. The problem is entirely about what compaction keeps.
✦ Why this tool exists
The paper's own proposed mitigation is called Constraint Pinning — a training-free method that quarantines governance constraints from lossy compaction and restores violation to 0%. constraintguard is an open-source implementation of exactly that.

Corroborating & foundational
arXiv:2605.06445 · Dente, Satriani & Papotti · May 2026

Constraint Decay: The Fragility of LLM Agents in Backend Code Generation

Capable agents lose ~30 points of assertion pass-rate as structural requirements accumulate — reliability degrades exactly as the constraints pile up.
arXiv:2307.03172 · Liu et al. · Stanford & collaborators

Lost in the Middle: How Language Models Use Long Contexts

The foundational result: models use information at the start and end of a long context far better than the middle — so buried rules are the first to go unheeded.

Scenario

Declare constraints in a fenced ```constraints block using must: / should: lines (with an optional [id]). extract lifts them into a clean, validated JSON set — a false-positive-proof opt-in format.

Give it the original context and a compacted version. conformance extracts constraints from both and scores how many survived. Try the match mode — exact counts a constraint as dropped if its text changed; id only checks the id.

✕ Dropped (rotted)

pin re-injects the original constraints into the compacted context as a clean ```constraints block — guaranteed to round-trip, so the very next extract reads them back intact. This is the fix.

This is the ConstraintRot benchmark — the source of the 46% headline. It runs conformance over three realistic before/after fixtures right now, in your browser, and aggregates the drop. Toggle match mode to see the coding scenario's reworded rule count as dropped (exact) or surviving (id).

Quickstart — adopt it in a minute

Three ways practitioners use it.

A CLI for one-off checks, a library for your own pipeline, or a hook that closes the loop automatically on every compaction.

① CLI & CI gate

Score how many constraints survived a compaction — fail the build below a threshold.

# no install needed
npx constraintguard \
  conformance old.md new.md \
  --threshold 0.9 --json

② Library

Extract, pin, and score inside your own agent runtime. Zero dependencies, pure ESM.

import { pinConstraints }
  from 'constraintguard';

// re-inject before the next turn
ctx = pinConstraints(set, ctx);

③ Auto-pin hook

A drop-in Claude Code hook: extract on compaction, re-pin right after. Self-applying.

// .claude/settings.json
"PreCompact": [{ "hooks": [
  { "command":
    "cg hook pre-compact" }]}]
