The problem — "ConstraintRot"

Compaction is lossy, and constraints are the first casualty.

To fit a growing conversation into a fixed context window, agents summarize their own history. That summary keeps the gist — and quietly drops the rules. The research (arXiv 2606.22528) measured violation rates climbing from 0% to as high as 59% after a compaction. constraintguard reproduces it, then fixes it.

Scenario

Declare constraints in a fenced ```constraints block using must: / should: lines (with an optional [id]). extract lifts them into a clean, validated JSON set — a false-positive-proof opt-in format.

Give it the original context and a compacted version. conformance extracts constraints from both and scores how many survived. Try the match mode — exact counts a constraint as dropped if its text changed; id only checks the id.

pin re-injects the original constraints into the compacted context as a clean ```constraints block — guaranteed to round-trip, so the very next extract reads them back intact. This is the fix.

This is the ConstraintRot benchmark — the source of the 46% headline. It runs conformance over three realistic before/after fixtures right now, in your browser, and aggregates the drop. Toggle match mode to see the coding scenario's reworded rule count as dropped (exact) or surviving (id).

The OTel bridge maps a constraint set to a flat set of OpenTelemetry span attributes — attach them to any span and "which constraints were in force / dropped" shows up in your traces (Arize, LangSmith, Braintrust…). Dependency-free: it produces the attribute shape, you attach it with whatever exporter you already use.

What's in the box

Four commands, three adapters, one hook.

cg validate

Schema

Validate a constraint set against the open JSON shape {id, text, severity}.

cg extract

Extract

Lift declared constraints out of any context or session transcript.

cg pin

Pin

Re-inject constraints so they survive the next compaction. Round-trip guaranteed.

cg conformance

Conformance

Score retention between two contexts and list exactly what rotted.

--harness …

Adapters

Read constraints straight from Claude Code, Codex, and Cursor sessions.

cg hook

Compaction hook

A drop-in Claude Code hook that auto-re-pins on every compaction. Self-applying.
